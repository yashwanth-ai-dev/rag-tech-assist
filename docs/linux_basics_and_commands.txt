LINUX BASICS & COMMAND MASTERY — COMPLETE TROUBLESHOOTING GUIDE FOR IT SUPPORT

INTRODUCTION
Linux is widely used in servers, cloud systems, DevOps tools, firewalls, and enterprise backend systems. Every IT support engineer must understand the fundamentals. This document covers essential commands, tools, logs, permissions, services, processes, networking, and troubleshooting techniques.

----------------------------------------------------------------------------------------------------
SECTION 1 – UNDERSTANDING LINUX FILESYSTEM
----------------------------------------------------------------------------------------------------

LINUX DIRECTORY STRUCTURE
/               → root of the file system  
/bin            → essential binaries  
/sbin           → admin binaries  
/usr            → user apps and libraries  
/etc            → configuration files  
/home           → user directories  
/var            → logs, temporary data  
/tmp            → temporary files  
/opt            → optional software  
/run            → service runtime data  
/dev            → devices  
/proc           → process information  
/lib, /lib64    → system libraries  

KEY RULES:
- Everything is a file.
- Linux is case-sensitive.
- Config files live in /etc.

SECTION 3 – PERMISSIONS & OWNERSHIP
----------------------------------------------------------------------------------------------------

FORMAT:
-rwxr-xr--  
1st part → file type + permissions  
2nd → owner  
3rd → group  
4th → others

PERMISSION VALUES
r = 4  
w = 2  
x = 1  

COMMON PERMISSIONS:
chmod 755 file  
chmod 644 file  
chmod -R 777 folder (DANGEROUS)

OWNERSHIP
chown user file  
chown user:group file  
chgrp group file

CHECK USER & GROUP:
whoami  
groups  
id

DELETE USER:

----------------------------------------------------------------------------------------------------

Most logs are under:

SECTION 13 – COMMON TROUBLESHOOTING SCENARIOS
----------------------------------------------------------------------------------------------------

SCENARIO 1 – HIGH CPU USAGE  
Steps:
1. Check processes → top  
2. Identify culprit  
3. Restart service  
4. Update or reconfigure app  
5. Check logs

SCENARIO 2 – SERVER CANNOT REACH INTERNET  
Steps:
1. Check IP → ip a  
2. Check DNS → resolv.conf  
3. Ping gateway  
4. Ping 8.8.8.8 → tests ISP  
5. Check firewall rules  
6. Check proxy settings  

SCENARIO 3 – APPLICATION NOT STARTING  
Steps:
1. systemctl status app  
2. journalctl -u app  
3. Check config file syntax  
4. Check logs in /var/log  
5. Restart service  

SCENARIO 4 – PERMISSION DENIED  
Steps:
1. Check file owner → ls -l  
2. Check chmod  
3. Check group membership  
4. Check SELinux (if RHEL):  
   getenforce  

SCENARIO 5 – SSH NOT WORKING  
Steps:
1. Check service: systemctl status ssh  
2. Check firewall  
3. Check port  
4. Check permissions of ~/.ssh  
5. Check host key issues
